Политика конфиденциальности Mengram

Версия 1.1 | Последнее обновление: 30 апреля 2026

1. Предмет и область применения

Настоящая Политика конфиденциальности (далее "Политика") применяется к мобильному приложению "Mengram" (далее "Приложение"), разработанному и распространяемому под лицензией GNU General Public License версии 2.0 или более поздней версии (далее "GPL v2.0 or later").

Приложение представляет собой модифицированный клиент протокола Telegram, адаптированный для функционирования в условиях сетевых ограничений посредством встроенного механизма проксирования трафика (далее "Встроенный прокси").

2. Сбор и обработка данных разработчиком

2.1 Исходное положение

Разработчик приложения (далее "Разработчик") не собирает, не логирует, не анализирует и не передает никакие персональные данные пользователей. Приложение не содержит компонентов аналитики, телеметрии, трекинга или сбора статистики.

Важное уточнение по IP-адресам: Разработчик не имеет доступа к IP-адресам пользователей. IP-адрес пользователя может быть виден только конечными узлами (MTProto-прокси или серверами Telegram) в процессе установления соединения, что технически необходимо для работы сети Интернет и не контролируется Разработчиком Mengram.

2.2 Открытый исходный код

Весь исходный код Приложения размещен в открытом доступе на платформе GitHub (github.com/Rtopix/Mengram) под лицензией GPL v2.0 or later. Пользователь имеет право и может самостоятельно проверить отсутствие любых механизмов сбора данных.

3. Telegram как основная платформа

Приложение является клиентом Telegram и использует инфраструктуру сервиса Telegram для:

  • Передачи и хранения сообщений
  • Синхронизации контактов пользователя
  • Управления учетной записью
  • Доставки медиаконтента
  • Обработки других данных пользователя

Обработка персональных данных на серверах Telegram регулируется Политикой конфиденциальности Telegram и не входит в компетенцию Разработчика Mengram.

4. Встроенный WebSocket прокси и MengramProxyEngine

4.1 Архитектура

Приложение включает встроенный механизм проксирования трафика (далее "Встроенный прокси"), который функционирует локально на устройстве пользователя на локальном адресе 127.0.0.1, порт 1443.

Встроенный прокси основан на следующих открытых проектах:

  • Оригинальный проект: tg-ws-proxy © 2026 Flowseal (github.com/Flowseal/tg-ws-proxy)
  • Android адаптация: tg-ws-proxy-ANDROID © 2026 ShadowLev / LemoLev (github.com/LemoLev/tg-ws-proxy-ANDROID)

Реализация на языке Go с использованием Android NDK, интегрирована в Mengram на условиях их исходных лицензий.

4.2 MengramProxyEngine - механизм поиска проксей

Приложение включает компонент MengramProxyEngine (далее "Механизм"), который выполняет следующие функции:

  • Автоматический поиск доступных MTProto прокси-серверов из открытых репозиториев GitHub
  • Проверка живучести найденных прокси-серверов посредством простых TCP проверок
  • Ротация между доступными прокси при разрыве соединения
  • Фильтрация прокси-серверов по критерию "доменное имя" (исключение числовых IP адресов)
  • Автоматическое переключение при недоступности текущего прокси

4.3 Источники прокси-серверов

Механизм получает список прокси-серверов из следующих открытых источников:

  • Открытые репозитории GitHub, содержащие списки MTProto прокси
  • Регулярно обновляемые публичные источники
  • Доменные имена CloudFlare (в случае их доступности)
  • Другие публично доступные MTProto прокси-серверы

Важно: Источники прокси могут меняться и обновляться в зависимости от доступности сервисов.

4.4 Принцип работы прокси

Встроенный прокси:

  • Функционирует исключительно на локальном устройстве пользователя
  • Не передает данные Разработчику или третьим лицам, связанным с Разработчиком
  • Не логирует содержимое сообщений или метаданные пользователя
  • Не сохраняет историю соединений
  • Используется исключительно для трансляции зашифрованного трафика Telegram через прокси-серверы при наличии сетевых ограничений
  • Не нарушает и не обходит механизмы сквозного шифрования Telegram (E2E)

4.5 Безопасность прокси-соединения

Все соединения через встроенный прокси защищены:

  • TLS/SSL шифрованием транспортного уровня
  • MTProto криптографическим протоколом Telegram
  • Сквозным шифрованием для приватных чатов
  • FakeTLS маскировкой для обхода DPI-фильтрации (если применимо)

5. Разрешения устройства

Как клиент Telegram, Mengram запрашивает следующие разрешения на устройстве:

  • Доступ к камере: Необходим для совершения видеозвонков и отправки фотографий
  • Доступ к микрофону: Необходим для голосовых звонков, видеозвонков и голосовых сообщений
  • Доступ к контактам: Необходим для синхронизации контактов и поиска друзей в Telegram
  • Доступ к хранилищу: Необходим для сохранения и отправки файлов, фото и видео
  • Доступ к геолокации: Необходим для отправки местоположения в чаты (опционально)

Пользователь может управлять этими разрешениями в Настройках устройства → Приложения → Mengram → Разрешения.

6. Права пользователя

Поскольку Разработчик не собирает никакие данные, вопросы защиты данных и реализации прав пользователя находятся в компетенции:

6.1 В отношении Telegram

  • Доступ к данным: Настройки → Приватность → Экспортировать данные
  • Удаление аккаунта: my.telegram.org → Настройки → Приватность → Удалить аккаунт
  • Запрос удаления данных: my.telegram.org
  • Управление контентом: Удаление сообщений, медиа, контактов доступно в приложении
  • Отключение аналитики: Настройки → Приватность и безопасность → отключить аналитику

6.2 В отношении Mengram

  • Удаление локальных данных: Удаление приложения полностью очищает все локальные кэши
  • Отключение прокси: Механизм прокси может быть полностью отключен в Настройках
  • Проверка исходного кода: Полный исходный код доступен для аудита на GitHub

7. Третьи стороны

7.1 Telegram

Обработка данных на серверах Telegram регулируется Политикой конфиденциальности Telegram. Разработчик Mengram не имеет контроля над обработкой данных на серверах Telegram.

7.2 Прокси-серверы - важное предупреждение

⚠️ Важно: При использовании встроенного прокси пользователь должен осознавать, что владельцы публичных прокси-серверов могут видеть следующую техническую информацию:
  • Ваш реальный IP-адрес
  • Факт того, что вы пользуетесь Telegram
  • Объем трафика (хотя не содержимое сообщений)
Однако содержимое сообщений остается защищенным благодаря сквозному шифрованию Telegram (E2E). Владелец прокси не может прочитать ваши сообщения, только видеть факт использования Telegram.

При использовании встроенного прокси трафик может маршрутизироваться через различные прокси-серверы, включая:

  • CloudFlare (политика: cloudflare.com/privacy)
  • Другие публичные MTProto прокси-серверы
  • Телеграм датацентры

Разработчик не контролирует политики конфиденциальности третьих сторон.

8. Безопасность и криптография
